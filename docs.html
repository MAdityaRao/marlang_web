Language Docs

Complete reference for the Mar programming language — syntax, types, classes, the compiler pipeline, and everything in between.

Introduction

#introduction

Mar is a compiled, statically-typed programming language written in C. You write clean .mar files and run them with a single command. Behind the scenes, Mar translates your code to C and invokes gcc or clang automatically.

The result: Fast native programs with a syntax that does not get in your way.

Design Philosophy

Semicolons are noise. Verbose for loops are boilerplate. Manual gcc invocations are friction. Mar eliminates all three without sacrificing performance. The compiler translates to idiomatic C, so you inherit every optimization GCC and Clang provide — for free.

Quick Start

#quickstart

Create a file called hello.mar with the following content:

hello.mar
int main() { print("Hello, World!\n") return 0 }

Run it:

mar run hello.mar
output
Hello, World!

That is the entire workflow. No separate compile step, no intermediate files to manage, no manually invoking gcc.

What happens under the hood

mar run yourprogram.mar does four things in sequence: parses the source into an AST, generates C code internally, invokes gcc or clang to compile it, then runs the resulting binary. The intermediate C file is temporary and cleaned up automatically.

Variables

#variables
Declaration and assignment
int x = 5 float ratio = 3.14 char letter = 'A' bool active = true
Reassignment
x = 10 x += 3 x -= 1 x *= 2 x /= 4

Variables must be declared before use. There is no implicit type casting — assign compatible types only.

Functions

#functions

Functions are declared with a return type, name, parameter list, and a block body. The main function is the entry point.

Function declaration
int add(int a, int b) { return a + b } int main() { int result = add(3, 4) print("%d\n", result) return 0 }
Void function (no return value)
void greet(int n) { print("Hello %d times\n", n) }

Functions must be declared before they are called. Recursive functions are fully supported.

Control Flow

#control-flow
if / else if / else
if(x > 0) { print("positive\n") } else if(x < 0) { print("negative\n") } else { print("zero\n") }

Braces are optional for single-statement bodies:

if(x > 0) print("positive\n")

Loops

#loops
while loop
int i = 0 while(i < 10) { print("%d\n", i) i = i + 1 }
for-range loop
for i in range(0, 10) { print("%d\n", i) }

range(a, b) iterates from a inclusive to b exclusive. The loop variable i is automatically declared as int. Under the hood it compiles to a C while loop for maximum performance.

break inside a loop
while(1) { int x = 0 take("%d", &x) if(x == 0) break print("got %d\n", x) }

Arrays

#arrays
Declaration with initializer
int arr[5] = {1, 2, 3, 4, 5}
Element access and mutation
arr[0] = 10 print("%d\n", arr[2])
Iterating an array with range
for i in range(0, 5) { print("%d\n", arr[i]) }

Array size must be a compile-time integer literal. Dynamic arrays are on the roadmap.

Input / Output

#io

Mar provides print() and take() as thin wrappers over C's printf and scanf.

print — formatted output
print("Hello, World!\n") print("x = %d, y = %d\n", x, y) print("pi = %.2f\n", pi)
take — formatted input
int n print("Enter a number: ") take("%d", &n) print("You entered: %d\n", n)

Classes & Objects

#classes

Mar supports object-oriented programming via the class keyword. Classes compile to C structs with associated functions. All objects are heap-allocated via the arena allocator.

Class definition
class Player { int health int score int kills void init(int h) { health = h score = 0 kills = 0 } void fight(int enemy_level) { health = health - enemy_level score = score + enemy_level kills = kills + 1 } }
Object creation and use
Player hero = new Player(100) hero.fight(5) print("HP: %d\n", hero.health)
How it compiles

class becomes a C typedef struct plus associated functions. new Player(100) becomes arena_alloc + Player_init(100). hero.field compiles to hero->field in C because all class instances are pointer types. The dot operator in Mar always means pointer dereference — you never have to think about ->.

The init method is required and is called automatically by new. Class names can be used as types in function parameters.

Switch Statement

#switch

Mar’s switch statement has no colons after case labels and automatically inserts break after each case unless you already wrote one. This eliminates C's notorious accidental fallthrough.

switch / case / default
switch(n) { case 1 print("one\n") break case 2 print("two\n") break default print("other\n") }

Compiler Pipeline

#pipeline

The Mar compiler is a classic multi-pass pipeline. Every .mar file passes through these stages in order:

StageSource FileInputOutput
Lexer src/lexer.c Source text Token stream
Parser src/parser.c Token stream AST (Abstract Syntax Tree)
Code Generator src/codegen_c.c AST C source file
C Compiler gcc / clang C source file Native binary
Key translations in codegen
for i in range(a, b) → while loop in C print(...) → printf(...) take(...) → scanf(...) switch without colon → valid C switch + auto break class Foo {} → typedef struct + Foo_method() new Foo() → arena_alloc + Foo_init() obj.field → obj->field

Error Messages

#errors

Mar collects errors from all stages and prints them with the source line and a caret pointing to the exact column.

Error TypeMeaning
LexerErrorInvalid character in source file
ParserErrorMalformed code structure — missing brace, unexpected token
SemanticErrorLogic error — undefined variable, break outside loop
CodegenErrorInternal compiler error during C output generation
Example — undefined variable
SemanticError at primes.mar:7:8 Undefined variable 'primme' 7 | if(primme(i)) | ^
Example — unexpected token
ParserError at hello.mar:3:5 Expected '}' but got 'return' 3 | return 0 | ^

Memory Model

#memory

Mar uses an arena allocator for all compiler internals and for objects created with new at runtime.

How the arena works

All memory — AST nodes during compilation, class instances at runtime — is allocated from a single contiguous region. When the program exits, one call frees the entire arena. There are no per-allocation free() calls anywhere. This makes the compiler fast and eliminates use-after-free and double-free bugs by design.

Objects created with new live for the lifetime of the program. This is intentional: Mar programs are typically short-lived native tools where arena allocation is the optimal strategy. A garbage collector or reference counting system is not planned.

Example — Hello World

#ex-hello
int main() { print("Hello, World!\n") return 0 }

Example — Prime Numbers

#ex-primes
int prime(int n) { if(n < 2) return 0 int i = 2 while(i * i <= n) { if(n % i == 0) return 0 i = i + 1 } return 1 } int main() { print("Primes from 2 to 20:\n") for i in range(2, 21) { if(prime(i)) print("%d ", i) } print("\n") return 0 }
output
Primes from 2 to 20: 2 3 5 7 11 13 17 19

Example — FizzBuzz

#ex-fizzbuzz
int main() { for i in range(1, 101) { if(i % 15 == 0) print("FizzBuzz\n") else if(i % 3 == 0) print("Fizz\n") else if(i % 5 == 0) print("Buzz\n") else print("%d\n", i) } return 0 }

Example — RPG Player (OOP)

#ex-classes
class Player { int health int score int kills void init(int h) { health = h score = 0 kills = 0 } void fight(int enemy_level) { health = health - enemy_level score = score + enemy_level kills = kills + 1 } } int main() { Player hero = new Player(100) for i in range(1, 6) { hero.fight(i) print("Fight %d: HP=%d Score=%d\n", i, hero.health, hero.score) } print("Total kills: %d\n", hero.kills) return 0 }
output
Fight 1: HP=99 Score=1 Fight 2: HP=97 Score=3 Fight 3: HP=94 Score=6 Fight 4: HP=90 Score=10 Fight 5: HP=85 Score=15 Total kills: 5

Example — Array Sum

#ex-arrays
int main() { int arr[5] = {10, 20, 30, 40, 50} int sum = 0 for i in range(0, 5) { sum = sum + arr[i] } print("Sum = %d\n", sum) return 0 }
output
Sum = 150

Adding Features

#adding-features

The compiler has a clean separation of concerns. Here is where each kind of change belongs:

What to changeWhere
Add a new keywordsrc/lexer.c → KEYWORDS[] array
Add a new single-char tokensrc/lexer.c → single-char switch at bottom of lexer_tokenize
Add new syntaxsrc/parser.c → case in parse_stmt()
Add a new AST nodeinclude/mar/ast.h → StmtKind or ExprKind enum + union
Change how something compiles to Csrc/codegen_c.c → case in emit_stmt() or emit_expr()
Add a new typeinclude/mar/ast.h → TypeKind enum
Change error messagessrc/error.c
Recommended reading order

Start with include/mar/ast.h to understand all data structures. Then read src/lexer.c, src/parser.c, src/codegen_c.c, and finally src/main.c to see how it all connects.
